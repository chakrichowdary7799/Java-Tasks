# High-Throughput Coupon & Discount Engine

An enterprise-grade, high-concurrency Coupon and Discount Engine built with Spring Boot 3.x, Java 17+, and MySQL. This engine leverages the **Strategy Pattern** for clean coupon type extraction, enforces strict transactional concurrency boundaries via database **Pessimistic Locking**, and prevents performance bottlenecks using optimized **Fetch Joins**.

---

## 🚀 Key Architectural Features
- **Concurrency Protection**: Uses `Pessimistic Write` locks to safely block double-redemptions during high-load checkout scenarios.
- **N+1 Query Resolution**: Minimizes database round-trips via explicit `LEFT JOIN FETCH` structural entity mappings.
- **Extensible Math Engine**: Easily expand capabilities by dropping custom business rules into the decoupled Strategy Pattern layout.

## 🏃 Building and Running the Application

### Option A: Via VS Code User Interface
1. Navigate to `src/main/java/com/enterprise/couponengine/CouponEngineApplication.java`.
2. Click the inline **Run** or **Debug** lens button located directly above the `main` method declaration.

### Option B: Via Embedded Terminal
Execute the standard Maven boot command wrapper from the project root folder:
```bash
mvn spring-boot:run
```

## 🧪 Operational Testing Guide (REST API Verification)

Once your application starts up on port `8080`, your `schema.sql` file will automatically seed your instance with active testing tokens: `SAVE20`, `PERCENT15`, and `BIGSPENDER`. Use the terminal snippets below to interact with your system.

### 1. Register a New Custom Coupon Rule
```bash
curl -X POST http://localhost:8080/api/v1/coupons \
     -H "Content-Type: application/json" \
     -d '{
       "code": "SUPER50",
       "type": "PERCENTAGE",
       "discountValue": 50.00,
       "expiryDate": "2030-12-31T23:59:59",
       "usageLimit": 100
     }'
```

### 2. Retrieve Specific Coupon Details
```bash
curl -X GET http://localhost:8080/api/v1/coupons/code/SUPER50
```

### 3. List All Coupons Registered in the System
```bash
curl -X GET http://localhost:8080/api/v1/coupons
```

### 4. Apply Cumulative Coupons During Checkout
Applies seeded coupons against an existing order basket (Order ID `1` was auto-generated by the database seed layer during startup):
```bash
curl -X POST http://localhost:8080/api/v1/checkout/apply-coupons \
     -H "Content-Type: application/json" \
     -d '{
       "orderId": 1,
       "couponCodes": ["SAVE20", "PERCENT15"]
     }'
```
